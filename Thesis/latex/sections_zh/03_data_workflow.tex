\section{实验数据与图像处理工作流}

\subsection{工作流概览}

图~\ref{fig:surrogate-data-flow} 展示了本文所用代理模型工作流的整体数据流，整个流程按四条泳道组织。
原始输入（高速 Mie 视频、标定配置和试验元数据）先经过 CUDA 加速的图像处理，生成一维特征和边界点；在此基础上，再得到基于二值化（BW）和基于 CDF 的穿深轨迹以及其他几何度量。
这些观测量通过无参数曲线拟合扩展为合成轨迹，并用于驱动三阶段神经网络训练：阶段~1（MSE）、阶段~2（NLL）和阶段~3（Raw + KD，知识蒸馏）。
最终训练得到的模型被部署到壁面碰撞筛查应用中，结合活塞几何和物理参数输出碰撞概率与可视化动画。
表~\ref{tab:workflow} 以文字形式总结了同样的阶段。
本章聚焦图~\ref{fig:surrogate-data-flow} 的前两条泳道：光学喷雾燃烧舱（OSCC）视频数据如何完成标定、预处理、配准并转换为逐羽流分辨的几何观测量。
轨迹目标构造、代理建模训练和概率筛查则留到下一章展开。

仅靠逐帧原始边界提取，并不能为第~4 章的代理模型提供其真正需要的东西：一个固定维度、带删失标记、逐羽流分辨、且能跨越不同孔数和帧率喷嘴进行比较的时序目标。
因此，本章把图像处理视为一层可审计的方法环节，而不是附属的预处理步骤；下文列出的五个方法学要素，正是把一张离散的、逐事件的观测表转换为上述结构化目标的关键所在。
已有喷雾图像分析往往报告单个工况下的穿深、锥角或边界描述符。
相比之下，本文后续代理模型需要跨喷嘴几何和工况、定义一致、逐羽流分辨且考虑删失的时序目标。
因此，本章填补的是从原始光学喷雾记录到可供机器学习使用的观测目标之间的构造缺口。

这些任务基于 Wärtsilä OSCC 的数据完成。
原始材料是 TB 级、室温、非反应、顶视 Mie 散射记录，在固定相机布置下跨多个工况和喷嘴几何采集得到。
整条链路先手动标定喷嘴几何，再通过 CUDA 加速预处理、极坐标配准和羽流条带提取生成逐羽流观测量，
并通过降阶拟合构造穿深目标，随后交给第~4 章所述的三阶段 MLP surrogate。
在实现层面，该图像处理阶段被构建为一条四泳道重叠流水线——磁盘预取、GPU 计算、CPU 特征提取与异步输出并发运行——并由可恢复的检查点合约管控，从而使多 TB 归档能够在单块工作站 GPU 上端到端重新处理，而非串行逐帧处理（第~\ref{subsec:concurrency-architecture}~节）。

\smallskip\noindent\textbf{本章贡献.} 本章为从图像到观测量的处理流水线贡献了五个可复用的方法学要素：
\begin{enumerate}
  \item 对数比（同态）前景估计，将眩光与非均匀照明视为乘性效应，并在对数域中以加性偏置将其去除（第~\ref{subsec:background-normalization}~节）；
  \item 秩一可分离的 Sobel 边缘增强，整段保持在 GPU 上、以两次一维卷积完成（第~\ref{subsec:edge-enhancement}~节）；
  \item 基于 FFT 谐波相位的对齐，把已知的多孔数转化为逐录像自动的羽流配准规则，而非逐录像手工设定参考角（第~\ref{subsec:plume-registration}~节）；
  \item 基于累积分布（CDF）的穿深前沿，对破碎的前缘像素稳健，同时与标准柴油喷雾穿深惯例保持一致（第~\ref{subsec:penetration-measurement}~节）；
  \item 四泳道重叠的 GPU 流水线及可恢复检查点合约，可在单块工作站 GPU 上端到端重新处理多 TB 归档（第~\ref{subsec:concurrency-architecture}~节）。
\end{enumerate}

\subsection{喷嘴几何的人工标定}\label{subsec:nozzle-calibration}

人工标定确定后续批量处理所需的喷嘴中心、标定半径和角度参考。
操作者在代表性帧中选择可见圆形喷嘴特征上的点。在数学上，这些选定的喷嘴边缘点 $(x_i,y_i)$ 用代数圆模型拟合：
\[
x_i^2+y_i^2 + D x_i + E y_i + F = 0,
\]
并通过标准线性求解器（例如 \texttt{np.linalg.lstsq}）以最小二乘形式求解 \cite{Pratt1987CircleFit,Taubin1991CircleFit}。恢复得到的参数为：
\[
c_x=-\frac{D}{2}, \qquad c_y=-\frac{E}{2}, \qquad r=\sqrt{c_x^2+c_y^2-F},
\]
其中 $(c_x, c_y)$ 是喷嘴中心， $r$ 是拟合半径。同一拟合还会根据所选边缘点相对于理想等间距集合的偏差，估计一个平均角度偏移。

标定值会写入配置文件，之后批量流程就可以在无需手动重新居中的情况下处理整个文件夹。最小的 \texttt{config.json} 模式是 \{\texttt{plumes}, \texttt{centre\_x}, \texttt{centre\_y}, \texttt{inner\_radius}, \texttt{outer\_radius}\};两个半径共同定义环形处理掩码、局部孔口原点和下游条带几何。

像素到毫米的比例并不是通过完整相机内参标定得到的，而是由已知的外环物理尺度反推：
\[
s_{\mathrm{mm/px}}=\frac{90.0}{r_{\mathrm{outer,px}}},
\]
其中 $r_{\mathrm{outer,px}}$ 是配置文件里保存的 \texttt{outer\_radius}。

光路（相机焦距、工作距离与传感器对准）由实验团队按照本次测试所用舱体几何的高速摄影常规做法配置；本文未对镜头畸变进行正式表征（例如棋盘格内参标定），该归档中也没有相应记录。由于轴向测量兴趣区（$<\SI{90}{mm}$）相对于相机到羽流的工作距离较小，喷雾三维锥形展开所引入的离面视差预期相对于上文反算得到的固定像素-毫米比例是二阶效应；该假设的定量验证留待第5章下游的 LONO 一致性检验。

对于每段记录，处理链需要回答五个实际测量问题，下文各小节分别专门处理其中一个：喷嘴中心在哪里（第~\ref{subsec:nozzle-calibration}~节）、每条羽流方向在哪里（第~\ref{subsec:plume-registration}~节）、如何把每条羽流表示到共同的局部坐标系中（第~\ref{subsec:binarization-repair}~节及其之前的仿射重映射）、液相前沿已经穿透多远（第~\ref{subsec:penetration-measurement}~节），以及哪些边界描述符可以被一致提取（第~\ref{subsec:boundary-features}~节）。

对于每个 \texttt{.cine} 记录，处理流程会把视频加载到当前数组后端，复用整个文件夹的环形掩码，执行多孔预处理和后处理步骤，提取逐羽流穿深曲线，计算基于边界的几何度量，并导出同步 CSV 和元数据文件。
这条链中数值开销最大的部分都尽量保留在后端本地：当 CuPy 可用时，卷积、角度分箱、重映射、阈值处理、连通域清理和形态学操作都会沿着 CUDA 路径执行，直到最终表格导出为止。

这一实现选择源自早期内部流程的实际限制。旧流程依赖逐帧 MATLAB 处理，缺少有效批量并行：处理约 50~GB 视频通常要花一天左右，而且容易崩溃。
因此，把重型图像操作迁移到批量 CUDA 导向流程中，在本文里属于方法设计，而不是单纯的软件优化。
以下各小节所描述的全部算法阶段都在该批处理流水线内执行；使它们能在单块 GPU 上对整个归档端到端运行的并发设计——一种由可恢复检查点管控的四泳道重叠执行模型——将在其所调度的各项操作介绍完毕之后，于第~\ref{subsec:concurrency-architecture}~节详述。

\subsubsection{背景归一化与前景提取}\label{subsec:background-normalization}

批量流程中的第一个算法贡献，是为噪声较强的 Mie 视频而设计的前景构建阶段，而不是面向理想二值图像的方案。
其目标是在任何几何测量之前，在非均匀照明下估计稳定的相对前景对比。
在对数比前景提取步骤中，原始帧 \(V_t(x,y)\) 映射为
\[
L_t(x,y)=\log(V_t(x,y)+\varepsilon),
\qquad
L_{\mathrm{bkg}}(x,y)=\operatorname{median}_{t<t_{\mathrm{SOI}}} L_t(x,y),
\]
随后进行对数比重建：
\[
R_t(x,y)=\exp\!\left(L_t(x,y)-L_{\mathrm{bkg}}(x,y)\right)-1
=\frac{V_t(x,y)-B(x,y)}{B(x,y)+\varepsilon}.
\]
下一步将 \(R_t\) 乘以强度门 \(M_t(x,y)=\mathbf{1}[V_t(x,y)>\alpha B(x,y)]\)，减去喷射前基线，并施加软阈值
\[
F_t(x,y)=\max\!\left(M_t(x,y)R_t(x,y)-R_0(x,y)-\tau,\,0\right),
\]
其中 \(\varepsilon\) 是防止对数零的很小常数，\(t_{\mathrm{SOI}}\) 是喷射起始时刻，\(B(x,y)=\exp(L_{\mathrm{bkg}}(x,y))-\varepsilon\) 是线性尺度背景，\(\alpha\) 是强度门的对比乘子，\(R_0(x,y)\) 是喷射前残余基线，而 \(\tau\) 是用于抑制背景噪声的软阈值。
随后再用稳健分位数对结果重新缩放，然后才进入几何处理。
该对数形式起到同态比值滤波的作用：空间眩光、渐晕和背景照明被视作乘性图像效应，并在对数空间中转化为可相减的加性项。
这一序列对 Mie 视频很重要，因为原始强度并不是液体体积分数的直接度量，而实际目标是在非均匀照明、眩光和背景纹理下稳定提取几何，而不是做精确光学反演 \cite{BohrenHuffman2004,Linne2013SprayImaging,GonzalezWoodsDIP}。

\subsubsection{基于梯度的边缘增强}\label{subsec:edge-enhancement}

第二个贡献是直接作用于整个视频立方体的显式边缘增强高通阶段。
其物理目的，是在逐羽流分割前突出局部液相边界对比。
Sobel 核 \(K_x\) 和 \(K_y\) 只生成一次，并通过二维卷积应用：
\[
S_t(x,y)=\sqrt{\left(K_x * F_t\right)^2+\left(K_y * F_t\right)^2},
\]
其中，\(K_x\) 和 \(K_y\) 分别是水平和垂直方向的 Sobel 卷积核，\(*\) 表示二维卷积算子，\(F_t\) 是预处理后的前景图像，\(S_t\) 则是得到的梯度幅值。

为了保证批量处理时的计算效率，$K_x$ 和 $K_y$ 的二维卷积被分解为顺序的两次一维卷积。由于方向 Sobel 卷积核是秩一矩阵，它们可以表示为一维算子的外积：
\[
K_x = \mathbf{g} \otimes \mathbf{d}_g, \qquad K_y = \mathbf{d}_g \otimes \mathbf{g},
\]
其中 $\mathbf{g}$ 是离散高斯包络， $\mathbf{d}_g$ 是其导数。将二维卷积分解为两个顺序的一维卷积后，对于 $k\times k$ 尺寸的算子，每个像素的乘加计算复杂度从 $O(k^2)$ 降低到 $O(k)$。在 GPU 后端中，这些一维卷积直接通过 CuPy 库函数（\texttt{cupyx.scipy.ndimage}）进行调度，从而在设备数组上高效执行两次 1D 通道，避免了手写 CUDA 核函数及自定义共享显存分块的开销。

随后再次进行稳健缩放，并施加环形几何掩码。
这一步在处理链中很关键，因为它把低对比度灰度喷雾转化为边缘主导的表示，同时让整个操作都停留在 CuPy 后端。
这里的贡献不在于 Sobel 算子本身，而在于把它集成进一条避免反复主机--设备拷贝的批量视频工作流中，为每个羽流准备一致的下游几何处理 \cite{GonzalezWoodsDIP,Szeliski2010Vision}。

\subsubsection{基于角度强度剖面的羽流方向估计}

预处理完成后，流程不再通过笛卡尔坐标中的轮廓追踪寻找羽流方向。
相反，处理后的图像会围绕已标定的喷嘴中心重参数化到极坐标中。
对于每个像素，
\[
\theta(x,y)=\operatorname{atan2}(y-c_y,\;x-c_x)\bmod 360^\circ,
\]
每帧在角度箱 \(k\) 中的角度信号累积为
\[
A_t(k)=\sum_{(x,y)\in \mathcal{B}_k} I_t(x,y), \qquad
D_t(k)=\frac{A_t(k)}{\#\mathcal{B}_k},
\]
其中 \(\mathcal{B}_k\) 是角度落入箱 \(k\) 的像素集合，\(I_t(x,y) = F_t(x,y)\) 是扣除背景后的前景强度（定义见第~\ref{subsec:background-normalization}~节），\(A_t(k)\) 是累积角度信号，\(D_t(k)\) 是平均角度密度，\(\#\mathcal{B}_k\) 则是该箱中的像素数。
在 CuPy 实现中，角度分箱和加权求和都直接在 GPU 数组上完成。
概念上，这是把 2D 图像径向压缩成一个 1D 角度占用信号；对多孔对齐而言，它比原始帧中的边缘碎片稳定得多 \cite{Szeliski2010Vision}。

\subsubsection{多孔喷雾的傅里叶取向对齐}\label{subsec:plume-registration}

随后不是从固定偏移量出发，而是从角度信号的周期性中估计羽流导向角。
设
\[
\bar{A}(k)=\sum_t A_t(k)
\]
为时间求和的角度剖面，设 \(P\) 是已知羽流数目。
为了对齐羽流取向，基于快速傅里叶变换（FFT）的相位对齐步骤计算 \(\bar{A}\) 的实 FFT，读取羽流谐波 \(P\) 的相位，并将其转换为角度偏移：
\[
\hat{\phi}_0
=
-\frac{\arg\!\left(\mathcal{F}\{\bar{A}\}[P]\right)}{P}\frac{180^\circ}{\pi}.
\]
这正是对齐具有自适应性的原因：该方法使用当前数据集中观测到的谐波相位，而不是为每条记录手动假定一个参考角。
多孔处理流程随后构造羽流方向
\[
\phi_p=\frac{360^\circ p}{P}-\hat{\phi}_0, \qquad p=0,\dots,P-1,
\]
并结合三角阈值占用掩码和短间隙填充来排除空角扇区。
FFT 本身是标准方法；这里的贡献在于利用已知喷油器多孔数，把角度密度信号转成一条自动羽流配准规则 \cite{CooleyTukey1965FFT,Szeliski2010Vision}。
该配准流程对孤立的单孔异常在结构上是稳健的：相位估计所依据的信号已经在整段帧数和径向坐标上求和平均，因此单个故障孔对聚合谐波相位的影响有限。持续性的液压异常（如喷孔堵塞）在本次测试中未被观察到；更重要的是，拟合流程下游所应用的稳健离群点与质量控制过滤（第~\ref{sec:SGQR-target-construction-en}~节）被专门设计用来标记并剔除表现出异常贯穿距离行为的轨迹，即便配准本身受到轻微扰动，该机制也能拦截这类失效模式。

同一个 occupied-angle mask 还可以给出一个简单的锥角代理量。
做法是把 bin-wise 掩码视为定义在 \([0,360^\circ)\) 上的周期信号，把跨越 \(0^\circ/360^\circ\) 边界的 \texttt{True} 段合并，再把每个连续段长度从 bins 转成角度。
得到的角宽量化了每条羽流在整段记录中所占据的角度支持范围。
我们同时报告总占用角和连续段平均宽度；后者可以作为一个粗略的锥角代理，因为它无需逐帧单独拟合边缘，就能概括羽流的平均角向展开宽度。

\subsubsection{重映射到羽流对齐条带坐标}

一旦得到目标角度，每条羽流都会被重映射到一个共同的喷嘴左侧条带坐标系中。
其物理目的，是把所有羽流测量表达在共享的局部轴线上。
从图像处理角度看，每个羽流角 \(\phi_p\) 定义了一个仿射旋转加平移变换，把喷嘴放到输出条带的左边缘和中间高度；重映射通过以下逆关系完成：
\[
\begin{bmatrix}
x_{\mathrm{src}}\\y_{\mathrm{src}}
\end{bmatrix}
=
\mathbf{A}^{-1}
\!\left(
\begin{bmatrix}
u\\v
\end{bmatrix}
-\mathbf{b}
\right),
\qquad
\mathbf{A}
=
\begin{bmatrix}
\cos\phi_p & \sin\phi_p\\
-\sin\phi_p & \cos\phi_p
\end{bmatrix},
\qquad
\mathbf{b}=\mathbf{t}-\mathbf{A}\mathbf{c},
\]
其中 \(\mathbf{c}=(c_x,c_y)^\top\) 是喷嘴中心，\(\mathbf{t}\) 是条带原点平移。
对每条羽流，逆采样坐标图只需从正向仿射变换中预先计算一次，随后用最近邻、双线性、双三次或 Lanczos 核对整个视频立方体进行重映射和重采样。
这些图保存的是源图像采样坐标，而不是喷雾强度：X 图像素值给出输出条带像素在源图像中采样的 \(x\) 坐标，Y 图则保存对应的 \(y\) 坐标。
多孔流程在重映射后应用固定的标准化条带尺寸和三角形羽流掩码。
每个羽流角只需预计算一次逆映射，因此逐帧开销与旋转无关；后续所有度量都在同一个局部条带坐标系里测量 \cite{Szeliski2010Vision}。

\subsubsection{羽流条带二值化与形态学修复}\label{subsec:binarization-repair}

\begin{figure}[t]
\centering
\includegraphics[width=0.92\linewidth]{fig_threshold_mask_repair_summary.pdf}
\caption{从高通表示到二值喷雾支持的紧凑示例。（a）整个视频环形区域内的高通强度分布，并标出固定支持阈值；（b）代表帧上的单羽流高通条带；（c）固定阈值化得到的原始二值支持；（d）形态学修补后的二值支持，并高亮被添加和移除的像素。}
\label{fig:threshold-mask-repair-summary}
\end{figure}

高通条带不再使用 triangle threshold 进行二值化，因为该规则对接近零的峰值、长尾强度分布以及掩码引入的零值比例都很敏感。
当前处理首先对每个羽流的 \((t,y,x)\) 高通条带体做基于百分位数的最小--最大归一化，把 \(q_{\min}=5\) 和 \(q_{\max}=99.9\) 百分位映射到 0 和 1，并截断范围外的值。
随后在归一化条带 $\tilde{S}_p(t,y,x)$ 上使用单一固定阈值 \(T_{\mathrm{BW}}=0.01\)：
\[
B_p(t,y,x)=\mathbf{1}\!\left[\tilde{S}_p(t,y,x)>T_{\mathrm{BW}}\right].
\]
百分位归一化吸收了不同视频之间的亮度和局部对比度差异，而固定阈值只表示归一化后的边缘响应是否足够强。
图~\ref{fig:threshold-mask-repair-summary}(a) 中标出的阈值，就是归一化强度尺度上的这个固定支持阈值。

当二值支持需要进一步修补时，时间与逐帧二值支持修补步骤采用先时间体、后单帧的顺序。
对于每个羽流的 \((t,y,x)\) 二值体，流程先用 \(3\times3\times3\) 结构元做 closing，再填孔并保留三维最大连通域；随后对每一帧再保留二维最大连通域。
前一步利用时间邻接性修补短暂断裂，后一步去除每帧中的悬挂碎片。
图~\ref{fig:threshold-mask-repair-summary}(d) 展示了这一修补路径对单帧支持掩码的影响；它应被理解为二值几何度量的稳健化步骤，而不是对原始 Mie 强度场的物理恢复。



\subsubsection{基于轴向累积强度的穿深测量}\label{subsec:penetration-measurement}

穿深估计器刻意基于累积强度支持，而不是单个最亮像素或最外层的孤立团块。
对于每个羽流条带 \(I_p(t,y,x)\)，基于轴向累积强度的穿深估计首先沿横向坍缩，
\[
H_p(t,x)=\sum_y I_p(t,y,x),
\]
再对每一帧减去左边缘基线、截断负值，并通过自动阈值、最大连通域清理和二值开运算生成支持掩码。
在该支持区域内，轴向累积分布前沿计算构造轴向累积分布
\[
C_p(t,x)=\frac{\sum_{\xi \le x} \tilde{H}_p(t,\xi)}{\sum_{\xi} \tilde{H}_p(t,\xi)+\varepsilon},
\]
其中 \(\tilde{H}_p(t,\xi)\) 是扣除基线并清理后的横向强度和，\(\varepsilon\) 用于防止除零，而 \(C_p(t,x)\) 是得到的轴向累积分布。
随后，穿深前沿 \(\hat{x}_p(t)\) 被定义为满足
\[
\hat{x}_p(t)=\min \{x : C_p(t,x)\ge q\},
\]
的最小位置，其中分位数阈值在本文整个批量归档中固定为 \(q=0.995\)。
直观地说，这等价于把归一化后的轴向强度剖面视为液相沿喷雾轴向所在位置的一个分布：\(C_p(t,x)\) 即其累积分布，而穿深前沿 \(\hat{x}_p(t)\) 只是它的 \(99.5\%\) 分位数——即其后侧恰好累积了按列求和强度的 \(99.5\%\) 的那个轴向距离。分位数由分布的主体而非极端值决定，因此前沿之前一个孤立的亮斑点或脱离的散射光晕对累积和贡献甚微，无法把恢复出的尖端向外拉动；相比之下，仅取最亮或最外层像素的定义会被单个离群点劫持。
由于按列求和的强度 CDF 在前沿处急剧上升，恢复出的前沿在该取值的邻域内对 \(q\) 不敏感；\(q=0.995\) 被选作一个保守的前沿定义，用以排除液核之外微弱的散射光晕，而非作为调过的超参数。
随后减去内半径，把距离表达成相对于喷嘴环面的量，并进行显式轨迹清理。
这一步清理不是通过 cumulative maximum 把所有回退值抬高，而是在检测到第一个非物理后退跳跃后直接截断尾部：
\[
\Delta \hat{x}_p(j)=\hat{x}_p(j+1)-\hat{x}_p(j),\qquad
j_p^\star=\min\{j:\Delta \hat{x}_p(j)<0\},
\]
\[
\hat{x}_p(t)\leftarrow \mathrm{NaN}\quad \text{for}\quad t\ge j_p^\star .
\]
在这一处理规则中，截断从第一个负差分附近的帧开始，因此可能牺牲后面较弱的重新出现信号；这是一个保守选择，用来避免把孤立噪声尖峰或边界反射误认为真实的穿深回撤。
尾部截断之后，构建有效帧掩码，并去掉孤立的短有效区段。
这个 CDF 前沿是工作流在实践中的主要贡献之一：它对断裂的前缘像素远不如朴素的最大索引定义敏感，同时仍然给出与标准柴油喷雾用法一致的穿深量 \cite{Dent1971Penetration,HiroyasuArai1990,NaberSiebers1996}。

在第二段录像（下图）中，三种度量却一起攀升，并在共同的 $\approx 92$\,mm 上限处趋平——这并非穿深的物理终点，而是喷雾在喷射结束前就越过了有限的相机视场边界，使观测轨迹被截断而非物理饱和。此后本文将这一结构性机制称为\emph{视场右删失（FOV right-censoring）}。
这两种行为——后期信号丢失与视场删失——正是本节的显式轨迹清理以及随后基于删失感知的目标构造所要处理的结构。

\subsubsection{喷雾边界特征与等效体积描述符}\label{subsec:boundary-features}

二值化后的羽流条带还用于计算若干补充性的边界和几何观测量。
投影面积为
\[
A_p(t)=\sum_{x,y} B_p(t,y,x).
\]
轴向二值穿深通过占用像素数超过阈值的最右列得到，而极坐标穿深则从上边界点云和下边界点云定义为
\[
R_p(t)=\max\!\left(
\max_i \sqrt{x_{u,i}^2+y_{u,i}^2},
\max_j \sqrt{x_{l,j}^2+y_{l,j}^2}
\right).
\]
导出的观测量中还包括一个类体积的几何代理：
\[
V_p(t)=x_{\mathrm{scale}} \frac{\pi}{4}\sum_x \left(w_u(t,x)+w_l(t,x)\right)^2,
\]
其中 \(w_u\) 和 \(w_l\) 分别是上半宽和下半宽，而 \(x_{\mathrm{scale}}=1/\cos((180^\circ-\theta_{\mathrm{umb}})/2)\) 在需要时补偿伞角倾斜。
这个量不是三维重建，而是把二值投影截面解释成一个轴对称包络后得到的几何代理。
在量纲上 \(V_p\) 具有体积量纲；半宽与轴向坐标沿用第~\ref{subsec:nozzle-calibration}~节的毫米标度 \(s_{\mathrm{mm/px}}\)，故 \(V_p\) 以 \si{mm^3} 表示，作为相对几何描述量而非经标定的绝对喷射体积。
导出的描述符集合还包括上下界版本，
\[
V_{p,\max}(t)=\pi x_{\mathrm{scale}}\sum_x \max(w_u,w_l)^2,\qquad
V_{p,\min}(t)=\pi x_{\mathrm{scale}}\sum_x \min(w_u,w_l)^2,
\]
它们分别用两个半宽中的较大值和较小值作为局部有效半径，用于记录上下边界不对称时该代理的可能范围。
锥角代理通过两种方式计算：一是对局部边界角度取平均
\[
\alpha_{\mathrm{avg}}(t)=\overline{\arctan(y_u/x_u)}-\overline{\arctan(y_l/x_l)},
\]
二是拟合固定截距的边界线并计算
\[
\alpha_{\mathrm{lg}}(t)=\arctan(m_u)-\arctan(m_l).
\]
最后，靠近喷嘴的开启和关闭时刻由喷油器附近的一个小检测窗口推断得到。
检测逻辑对该窗口内的平均块强度进行鲁棒归一化，随后应用双阈值滞后方案（高阈值 0.50，低阈值 0.10），并标记检测到的有效喷射区间；图~\ref{fig:nozzle-timing} 展示了一条典型的时序信号及其液压延迟与喷嘴关闭事件。

这些度量之所以重要，是因为它们把每条对齐后的条带压缩成一个紧凑的几何状态向量；不过，本文的下游建模仍然以穿深为中心，因为在现有分析里它是支持最充分的目标 \cite{Linne2013SprayImaging,LefebvreMcDonell2017}。
综合来看，这些阶段——背景归一化、边缘增强、极坐标配准、条带重映射、二值化、穿深提取与边界描述符——必须对归档中全部 2{,}992 段录像逐一计算；第~\ref{subsec:concurrency-architecture}~节将回到流水线的组织方式，说明如何使其在单块工作站 GPU 上保持可行。

\subsubsection{大规模光学视频档案的计算处理}\label{subsec:concurrency-architecture}

要在单块 GPU 上端到端处理整个归档，关键是让该 GPU 持续保持忙碌，而不因磁盘读取或输出写入而停顿；本节所述设计把整段归档处理从日级、易崩溃的旧脚本缩短到 ${\approx}72$ 分钟（${\approx}42$ 段录像/分钟；表~\ref{tab:ch3-pipeline-timing}）。
流水线并行运行四条泳道：磁盘预取、GPU 计算、CPU 特征提取和文件输出（图~\ref{fig:ch3-pipeline-lanes}）。GPU 处理当前录像时，磁盘工作线程同步加载下一段；GPU 处理完成后，输出文件在后台异步写入；若处理中途中断，检查点文件记录当前进度，支持从断点处续跑。

视频载入采用深度为 1 的生产者—消费者预取：在 GPU 处理第 $N$ 条记录的同时，后台工作线程从磁盘读取第 $N+1$ 条，使 GPU 计算与磁盘 I/O 重叠而非串行执行。

主机到设备的数据传输与 CuPy 数组构造刻意保留在主线程，原因是 CuPy 的 CUDA 上下文与线程绑定；这使得核函数调度、同步和内存池状态均由同一线程统一管理，无需加锁。

文件输出由一个长驻三工作线程的异步 I/O 池负责：指标 CSV、元数据 JSON、边界归档和可选预览视频在 GPU 阶段结束后立即以 future 形式提交，从而使下一条记录的 GPU 处理与上一条的磁盘写出并行进行。

第四条泳道是与 GPU 阶段并发运行的 CPU 计算泳道：一旦某条记录的二值化羽流数据回传至主机，逐羽流边界和几何特征便在最多 $\lfloor n_{\mathrm{cores}}/2\rfloor$ 个线程组成的线程池中并行提取。由于该特征提取受 numpy/scipy 主导并会释放全局解释器锁（GIL），此处选用线程而非进程，可避免 Windows 多进程启动时复制大型羽流数组的开销。所有泳道共用同一套后端抽象：当 CUDA 环境可用时，重型操作分发到单卡 CuPy 执行；代码路径相同，在无 CUDA 环境下自动回退到 NumPy。载入时，每段视频还会被转换为 float16 并在设备上归一化，相对 float32 将工作集内存与显存带宽减半。

内存管理也与归档访问模式相匹配。

因为同一子文件夹内连续记录通常具有相同帧尺寸，保留池块可以让下一文件直接复用已经分配好的 GPU 缓冲区，避免反复往返 CUDA driver；而在子文件夹边界强制释放池块，则可以限制长时间归档运行中的碎片化。

这一元数据层级对混合的 Nozzle0（2022）与 Nozzle1--8（2024）归档尤其重要，因为它们的记录头约定不同。




\subsection{无像素级真值的边界提取}

本文的喷雾边界提取依赖梯度增强、形态学清洗和物理一致性校验。Mie 散射图像中，液滴散射产生强局部对比度，使喷雾与背景分界清晰，上述方案与这一成像特性相符。

本文未进行逐像素边界标注。对高速 Mie 时间序列进行此类标注需要大量人工工作，且所得标签反映的是标注者的主观判断，而非客观物理真值。喷雾边界随雾化和液滴破碎持续变化，不存在唯一的像素级真值。因此，需要参考掩码的 IoU 和 F1 等指标在此不适用。

本文通过叠加覆盖检查、时间连续性验证以及多个几何代理量之间的一致性来评估边界提取的可靠性。这一评估方式与本文目标相符：稳定提取贯穿距离、锥角和边界包络等可观测量，而非训练语义分割模型。
针对上述基于 CDF 的可观测量的定量下游证据——尤其是留一喷嘴（LONO）迁移诊断以及在上文所定义的视场右删失下的可控预测行为——在第~5~章呈现。
针对固定阈值和百分位数归一化参数（表~\ref{tab:ch3-preproc-params-zh}）的系统性敏感度扫描被认为超出本文范围；任何残余的分割漂移预期会以系统性偏差或第5章下游 LONO 迁移诊断中性能下降的形式间接显现，而非被直接刻画。
