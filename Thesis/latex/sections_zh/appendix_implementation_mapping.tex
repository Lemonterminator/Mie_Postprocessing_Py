\section{面向非机器学习读者的方法学简介}
\label{app:ml-primer-zh}

本附录面向以流体力学或机械设计为主要背景的读者，简要介绍本文涉及的四类技术：计算机视觉、鲁棒优化、GPU 并行计算与神经网络训练。正文只在需要解释方法选择时调用这些概念；更基础的术语说明集中放在这里，以避免文献综述承担教程功能。

\paragraph{计算机视觉与信号处理。}
计算机将图像存储为数值矩阵，每个像素对应矩阵中的一个强度值。图像处理本质上是在这些矩阵上执行数学运算。\emph{卷积} 是最常用的操作之一：一个小型模板窗口逐像素滑过图像，每一步执行加权求和。不同卷积核可用于平滑、边缘提取或纹理增强。Sobel 算子是一类用于估计局部亮度梯度的卷积核，对液相边界这类强度突变响应较强，因此适合喷雾边界定位。\emph{极坐标变换} 将图像从像素行列坐标转换为以角度和径向距离为轴的坐标系；围绕喷嘴中心使用该变换，可以把多孔喷雾的方向结构折叠为一维角度信号。\emph{快速傅里叶变换}（FFT）把信号分解为频率分量，使周期结构表现为频域峰值；本文用它识别多孔喷雾的旋转周期性并估计喷孔角度偏移。

\paragraph{鲁棒优化与曲线拟合。}
最小二乘拟合寻找一条曲线，使观测点到曲线的残差平方和最小。当少量帧受到光学噪声或量化误差影响时，平方惩罚会放大这些点的影响。Huber 损失在残差较小时保留平方惩罚，在残差超过阈值后切换为近似绝对值惩罚，从而降低离群点的杠杆效应 \cite{Huber1964,NocedalWright2006}。神经网络训练中的梯度下降则在当前参数处计算损失函数的偏导数，并沿降低损失的方向迭代更新参数。

\paragraph{GPU 并行计算。}
CPU 通常拥有较少但复杂的核心，适合串行逻辑；GPU 拥有大量较简单的核心，适合对许多数据元素同时执行相同运算。图像处理中的卷积、阈值化、角度分箱和逐帧聚合正好符合这种模式。CUDA 是 NVIDIA 的并行计算平台；CuPy 提供与 NumPy 相近的 Python 接口，使本文的处理链可以把主要数组运算转移到 GPU。其工程意义在于把 TB 级视频归档处理从难以反复迭代的长任务，转化为可随着标定、阈值和目标定义调整而重复运行的工作流 \cite{NickollsBuckGarlandSkadron2008}。

\paragraph{神经网络与深度学习。}
本文使用的多层感知机（MLP）由若干线性层和非线性激活函数组成，以工况特征和时间为输入，输出贯穿距离预测。Dropout 在训练中随机屏蔽部分神经元，降低稀疏工况支持下的过拟合风险 \cite{Srivastava2014Dropout}。AdamW 将权重衰减从自适应梯度更新中解耦，有助于改善泛化 \cite{LoshchilovHutter2019AdamW}。SiLU/Swish 等平滑激活函数使导数传递更连续，便于使用物理导数惩罚 \cite{Ramachandran2017Swish}。均方误差（MSE）对应条件均值回归；异方差高斯负对数似然（NLL）让网络同时输出条件均值和条件方差，为不同工况提供不同宽度的误差带 \cite{KendallGal2017}。知识蒸馏先训练一个在干净数据上表现稳定的教师模型，再用其软输出引导学生模型在标签稀疏或不可靠区域学习 \cite{HintonVinyalsDean2015}。
